Vue笔记

1.

在标签里使用mustache语法时，computed计算属性的函数可以不加（），但是如果是函数则必须加（），所以在某种程度上说，computed就是将函数转化成变量。

2.

let关键字声明变量的时候：

let sum=0,i=0;

let i,sum=0;

第一种i和sum都会定义且初始值为0，第二种方法则是i和sum都定义了，但是i没有初始值。

3.

用来比较或者判断两者是否相等，比较时可以自动转换数据类型。

　　=== 用来较为严格的比较，除判断数据是否相等 ，还会判断两者数据类型是否相同，如不相同也不会转换数据类型，返回（false）。

4.

这两句话最好放到代码的后面，否则就可能出现你界面做出来了，vue的数据没有渲染进去。

比如表格出来了，当里面的数据只显示一个

5.

splice（参数1，参数2，参数3）函数可以用来插入也可以用来删除

参数1：开始操作的位置

参数2：删除的个数，0则为不删除也就是添加。

参数3：只有在参数2 为0的时候使用，一般是数组[a,b,c]，意思就在参数1 的位置后面添加a，b，c三个元素。

6.

v-bind语法糖就是冒号， v-bind几乎可以绑定任何属性，甚至可以绑定class属性，同理也可以绑定id属性，用法都是一样的，都是直接在style标签里面.class{},#id{}然后书写样式，只是多了个绑定vue数据的步骤罢了。那么为什么要用v-bind呢
主要是要跟vue里面容器的数据互动，可以把v-bind是一根通进vue容器的绳子，这个绳子将html里的属性和
vue容器的里面的数据绑定起来。
我们都知道，a标签有个href属性，主要是保存链接，当我们点击a标签开始生效，那么当我们在vue里面定义了
一个url:’www.baidu.com'的时候，我们不能直接写在herf后面（<a href:”url”>），因为vue容器
是封锁的，不能直接去拿数据，这时候就要用到v-bind。

7.

v-for一定要与v-bind:key=”id”连用

8.

【面试】父子组件通信的方式有哪几种？

1 props $mit
2 回调函数(callback)
3 parent + $children
4 provide + inject
5 $attrs + $ilsteners
6 ref refs

props用于父传子，props写在子组件注册（Vue.component）里面，跟methods：，data：一样的位置

1、父组件可以使用 props 把数据传给子组件。
2、子组件可以使用 $emit 触发父组件的自定义事件。https://c.runoob.com/codedemo/5353/

9

ES6简写

const{XXX}=this.XXX

const{XXX}=this.props相当于const XXX = this.props.XXX

10

【报错】Maximum call stack size exceeded

解决：

items.splice(index ,1 ,items)->items.splice(index ,1 ,item)

错误代码:

methods:{
handleChange({index,value}){
const{items} = this
const item = items[index]
item.qty = value
items.splice(index ,1 ,items)

}

11

箭头函数：

其中脚手架vue build若选择的是runtime-only，那生成脚手架之后的src里的mian.js就有一个render：h => h(App)箭头函数

const materials = [
‘Hydrogen’,
‘Helium’,
‘Lithium’,
‘Beryllium’
];

console.log(materials.map(material => material.length));
// expected output: Array [8, 6, 7, 9]

//map()方法定义在JavaScript的Array中，它返回一个新的数组，数组中的元素为原始数组调用函数处理后的值。

栗有：

1 var arr = [“wei”,”ze”,”yang”];
2 arr.map(item=>”Mr.”+item); // [“Mr.wei”, “Mr.ze”, “Mr.yang”]
实则

1 var arr = [“wei”,”ze”,”yang”];
2 arr.map(function(item){
3 return “Mr.”+item;
4 });
